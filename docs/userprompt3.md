다음 기능을 전부 구체적으로 구현하라:

1. 메뉴 기능 히트박스 **프레스** 시 발생하는 팝업 메뉴. 여기서 캐릭터-히트박스의 통합 크기 조절(자동 맞춤), 앱 실행, 캐릭터 끄기, 터치스루 모드 on, (추가 예정)을 설정할 수 있다.
2. android 최근 정책 상 알림 모두 지우기에 의해 지워지지 않는 알림을 오버레이 중에 영구적으로 띄우는 기능 구현.
   - 이때, 알림은 모드에 따라 두 개로 나눈다. 터치 스루 모드일 경우, 클릭 시 터치스루 모드가 풀리는 기능. 터치 스루 모드가 아닌 경우 카카오톡과 비슷한 메세지/답장 구조의 알람 기능. 이때, 답장은 입력 가능해야함.
3. 선응답 기능
   - 이때, 선응답은 터치스루가 아닐때만 활성화됨
   - 메뉴-선응답 설정을 추가 후 해당에서, 선응답 빈도를 설정할 수 있음(a~b분 단위, 또한 onoff 추가)
   - 선응답 이벤트가 발생하면, api에 선응답용 프롬프트를 보내어 얻은 메세지를 그대로 알람으로 띄운다. 
   - 이때, 답장은 다시 api에 선응답용 프롬프트와 함께 바로 전달되어, 알람을 통한 지속적 대화가 가능하게 한다.
   - 메뉴-프롬프트 블록 편집에서, 저장 기능을 추가하여 해당 구조 그대로 저장하여 내보내기 불러오기 및 시스템 내에 프리셋으로 저장할 수 있는 기능을 추가한다.
   - 메뉴-api 관리 탭을 추가하여, 프리셋으로 저장되어 있는 api프리셋, 파라미터, 프롬프트 블록 프리셋을 각각 하나씩 묶어 하나의 출력 모델 프리셋으로 저장할 수 있도록 한다. 프리셋 목록에서 선응답용 출력 모델, 기본 출력 모델 두가지 역할을 지정할 수 있게 한다. 각 모델은 이름 그대로 앱에서 및 선응답시 사용할 api프리셋, 파라미터, 프롬프트 블록 프리셋이다.
