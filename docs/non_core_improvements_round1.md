# Non-core Improvements (Round 1)

이번 배치에서는 핵심 도메인 로직 변경을 최소화하고, UI/유틸/유지보수 편의 개선에 집중했습니다.

## 포함된 개선

- 공통 사용자 피드백 유틸 추가
  - `lib/utils/ui_feedback.dart`
  - 스낵바 표시, 클립보드 복사 + 안내를 공통화

- 공통 빈 상태 위젯 추가
  - `lib/widgets/empty_state_view.dart`
  - 채팅/설정 화면의 빈 상태 UI 일관성 개선

- 화면별 적용
  - ChatScreen
    - 빈 상태 공통 컴포넌트 적용
    - 메시지 롱프레스 `복사` 동작 실제 복사 수행
    - 명령 처리 `switch` 분기 명시화 (`break`)로 안정성 강화
    - 다크모드 친화 색상으로 일부 하드코딩 제거
  - ChatListScreen
    - 빈 상태 공통 컴포넌트 적용
    - 삭제 완료 안내 스낵바 공통 유틸 사용
    - 메뉴 분기 `break` 추가
  - SettingsScreen
    - API 프리셋 빈 상태 공통 컴포넌트 적용
    - 프리셋 저장 전 필수 입력 검증 추가
    - 저장 안내 스낵바 공통 유틸 사용
  - ThemeEditorScreen
    - 테마 이름 검증 에러 피드백 공통 유틸 사용
    - 미리보기 영역 색상 다크모드 대응 강화
    - 메뉴 분기 `break` 추가
  - MenuDrawer
    - 새 채팅 안내 스낵바 공통 유틸 사용
    - 표시 버전을 `pubspec` 버전과 정렬 (`1.0.0+1`)

## 의도적으로 하지 않은 것

- API 호출 흐름, 프롬프트 구성, 세션 저장 구조 같은 핵심 동작 변경
- 데이터 마이그레이션/스토리지 키 구조 변경
- 대규모 아키텍처 개편
