## 2) 블로커/실패 분석

### A. 공유 파일 충돌로 인한 병합 지연 (HIGH)
- 징후
  - 동일 파일에서 반복 충돌(2회 이상)
  - 재베이스 후 동작은 유지되나 UI 결과가 서로 상충
- 영향
  - 머지 순서 지연
  - 화면별 스타일 일관성 저하
- 우선 대응
  - 파일 오너십 즉시 고정(아래 3절)
  - 공용 컴포넌트 API 변경은 1인 최종 승인 후 반영

### B. 품질 게이트 반복 실패 (MEDIUM)
- 징후
  - 동일 원인으로 게이트 2회 이상 실패
  - 정책/정적 검사 항목에서 반복 리젝
- 영향
  - 기능 완료 상태여도 머지 불가
- 우선 대응
  - 실패 항목을 기능 수정과 분리해 핫픽스 단위로 처리
  - 문구/주석/형식성 정책 이슈를 우선 제거

### C. 시각 회귀 및 우선순위 역전 (MEDIUM)
- 징후
  - 색상/간격/빈 상태 변경이 화면 간 상충
  - 비핵심 polish가 안정화 항목보다 선행
- 영향
  - 사용자 체감 품질 저하
  - 리뷰 비용 증가
- 우선 대응
  - 시각 최종 결정권을 1인에게 집중
  - 기능성 개선(유틸/헬퍼) 머지 후 시각 패스 진행

### D. 비핵심 작업의 코어 경계 침범 (CRITICAL)
- 징후
  - `lib/services/*`, `lib/providers/*` 핵심 흐름 변경 포함
  - 스토리지 키/데이터 구조 변경 동반
- 영향
  - 본 레인 목표 위반
  - 회귀 범위 확대
- 우선 대응
  - 해당 변경은 즉시 분리
  - non-core PR에서 제거 후 별도 트랙으로 이관

## 3) 재배정 가이던스 (실행 기준)

### 재배정 트리거
- 4시간 이상 실질 커밋/진척 없음
- 동일 블로커가 2개 동기화 사이클(약 2시간) 이상 지속
- 동일 실패 원인으로 게이트 2회 이상 반복

### 파일 오너십(현재 사이클)
- Bolt 단독 오너
  - `lib/screens/settings_screen.dart`
  - `lib/utils/ui_feedback.dart`
- Nova 단독 오너
  - `lib/widgets/empty_state_view.dart`
  - `lib/screens/chat_list_screen.dart`
  - `lib/screens/theme_editor_screen.dart`

### 재배정 규칙
- Bolt 블로킹 시
  - Bolt는 설정/헬퍼 기능성 항목만 유지
  - 시각 정합 관련 잔여 변경은 Nova로 이동
- Nova 블로킹 시
  - Nova는 시각 일관성 항목만 유지
  - 헬퍼/폼 검증/안내 메시지 변경은 Bolt로 이동
- 양측 동시 블로킹 시
  - 신규 스코프 즉시 중단
  - 최소 단위 PR 2개로 분리(기능성 먼저, 시각 정합 후순위)

## 4) 머지 게이트 연동 지침

본 문서는 별도 머지 체크리스트를 대체하지 않고, 재배정 판단 기준만 제공한다.

머지 승인 전 필수 확인:
- non-core 범위 준수(코어 도메인 파일 변경 없음)
- 공유 파일 충돌 해소 및 오너 승인 완료
- 반복 실패 원인 제거 후 게이트 통과
- 롤백 가능한 최소 변경 단위 유지

## 5) 이번 사이클 즉시 실행안

1. 오너십 고정 후 교차 수정 금지
2. Utility/Convenience 라인 선머지(기능성 안정화)
3. UI/Design 라인은 선행 머지 기준으로 재베이스 후 머지
4. 미해결 블로커는 트리거 충족 시 즉시 재배정 실행
