# Phase 10: 히트박스 메뉴, 영구 알림, 선응답 기능

## 구현 범위

### 1. 히트박스 롱프레스 팝업 메뉴 (오버레이 내)
- 히트박스(투명상자) 롱프레스 → Native 팝업 메뉴 표시
- 메뉴 항목: 자동 맞춤 크기, 앱 실행, 캐릭터 끄기, 터치스루 모드 토글
- Native Kotlin → Flutter MethodChannel 양방향 통신

### 2. 영구 알림 시스템 (Foreground Notification)
- Android 정책 상 알림 모두 지우기에 의해 제거되지 않는 ongoing 알림
- 모드별 알림 분리:
  - 터치스루 모드: 클릭 시 터치스루 해제
  - 비터치스루 모드: 카카오톡 스타일 메시지/답장 알림 (Direct Reply)

### 3. 선응답 (Proactive Response) 시스템
- 터치스루 비활성 시에만 동작
- 설정: 빈도 (a~b분 간격), on/off
- 이벤트 발생 → API 호출 → 알림으로 표시
- 알림 답장 → API 재호출 → 지속 대화

### 4. 프롬프트 블록 프리셋 저장/불러오기
- 현재 프롬프트 블록 구조를 프리셋으로 저장
- 내보내기/불러오기 (JSON 파일)
- 시스템 내 프리셋 관리

### 5. API 관리 탭 (출력 모델 프리셋)
- API 프리셋 + 파라미터 + 프롬프트 블록 프리셋 = 출력 모델 프리셋
- 역할 지정: 기본 출력 모델, 선응답용 출력 모델

## 구현 순서
1. 데이터 모델 (선응답 설정, 출력 모델 프리셋, 프롬프트 블록 프리셋)
2. 서비스 레이어 (선응답 서비스, 알림 서비스)
3. Native Android (영구 알림, 히트박스 메뉴, 답장 처리)
4. Flutter UI (설정 화면들)
5. Provider 연결
