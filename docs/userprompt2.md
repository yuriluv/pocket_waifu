모바일 앱 개발 중(바이브 코딩)인데, 아래 문제가 있어.
이를 해결하기 위해, 코파일럿에게 영어로 제안서 프롬프트를 제공하여 해결하고 싶어. 구체적인 제안을 토대로 내 해결 요청을 수행할 수 있게.
제안서는 문제 원인 추정과, 해결 방안 제시로 구체적으로 구성해야해.

---

문제:
1. 오버레이 기능의 드래그 기능을 잠시 끄고 터치가 작동되지 않게 함(커스텀 히트박스 추가 목적) 근데, 앱 내에서는 정상적으로 LIVE2D가 터치되지 않고, 뒤에 있는 ui가 터치되나, 앱 밖에서 다른 앱 위에 그리기 상태에서는 여전히 live2d가 터치가 되어 다른 터치 상호작용을 하지 못함(live2d 터치 시 아무 반응 없고 아래 메세지만 출력됌/원래는 live2d가 투과되어 뒤에 있는 버튼 등이 상호작용되야 함:...는 최신 버전 안드로이드에 최적화 되어 있지 않습니다. 화면 터치가 지연되거나 인식되지 않을 수 있습니다)
2. 투명도 문제에 관하여, 원래 하얀 박스 내에 캐릭터가 있는 기본 구조에서, 적당한 방법으로 하얀박스만 투명하게 만듬. 근데, 캐릭터 투명도가 완전히 불투명한게 아니라 10%정도 투명함. 

해결 요청:
1. 앱 밖에서도 라이브 투디가 투과되어 터치가 정상적으로 되게 하기.
2. 투명도 문제를 해결하는 겸 기존의 기능과 연동하기. 기존의 표시 설정에 있는 기능인 투명도 조절 기능이 현재 배경 투명화 작업을 하면서 더 이상 기능하지 않게 됌. 이에, 해당 기능을 캐릭터 투명도 설정으로 하여, 작동되게 하면서, 캐릭터의 투명도를 자유롭게 조절 가능하게 바꾸기. 이때, 배경 투명도는 그대로 완전 투명하게.

현재 모바일 앱 구조(ai요약):


1) 투명도 구조 (캐릭터 vs 배경)
렌더 파이프라인 흐름

GLSurfaceView가 투명 Surface로 생성됨
EGL RGBA(8,8,8,8) + PixelFormat.TRANSLUCENT + setZOrderOnTop(true)
매 프레임 glClearColor(0,0,0,0) → 배경 알파=0으로 초기화
Cubism SDK DrawModel()이 캐릭터를 그려서 알파가 있는 픽셀을 쌓음
최종 Surface가 윈도우에 합성됨
현재 문제가 생기는 이유(구조상)

캐릭터가 “살짝 비침” → 캐릭터 픽셀의 알파가 1.0이 아님
(SDK 내부에서 프리멀티플라이드 알파, 마스크/블렌딩 상태가 섞이면서 알파가 낮게 남음)
overlayParams.alpha를 1.0으로 고정하면 원래 숨겨지던 알파 부족이 드러남
즉, 배경은 투명하게 유지되지만, 캐릭터 픽셀 알파가 낮아 보여서 비침 현상이 발생합니다.

2) 터치 구조 (앱 밖에서 터치 막힘)
오버레이 윈도우 구성

앱 내부: Flutter UI 위에 GLSurfaceView가 떠 있음
앱 밖(홈/다른 앱): 동일 오버레이가 시스템 위에 계속 떠 있음
터치 제어 흐름

WindowManager에 추가된 오버레이는 시스템 전체 터치 경로에 관여
FLAG_NOT_TOUCHABLE 이면 오버레이 자체는 터치를 받지 않음
하지만 특정 플래그/권한 조합이 있으면 시스템이 “다른 앱 터치 지연 경고”를 띄움
문제가 생기는 이유(구조상)

오버레이가 시스템 전역에 남아 있는 상태라서
시스템이 “터치 간섭 가능성”을 감지하고 경고/차단을 유발함
이는 FLAG_NOT_TOUCHABLE만으로는 완전히 해소되지 않는 경우가 있음
즉, 오버레이가 존재하는 것 자체가 외부 앱 터치에 영향을 줄 수 있습니다.
